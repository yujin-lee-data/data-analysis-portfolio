# 분석 기획서

> 이 문서는 프로젝트의 문제 정의, 분석 목적, 분석 범위와 기대 산출물을 정리합니다.

## 1. 프로젝트명

서비스 운영 데이터 품질 관리 프로젝트: 공개 이커머스 데이터 기반 주문·결제·배송·판매자 정합성 점검 및 오류 검수 우선순위 도출

## 2. 분석 배경

서비스 운영 지원 경험을 통해 서비스 운영에서 정보 정확성과 기준 확인이 중요하다는 점을 느꼈습니다. 상호나 사업자 정보처럼 단순해 보이는 내용도 정확하게 관리되지 않으면 결제 과정에서 혼선이 생기거나 서비스에 대한 신뢰가 낮아질 수 있다고 생각했습니다.

정보통계학을 전공하며 자료를 정리하고 기준에 맞춰 관리하는 작업을 자주 경험했습니다. 같은 데이터라도 정리 기준이 명확하지 않으면 이후 수정이 반복된다는 점을 여러 과제를 통해 체감했습니다. 실제 업무에서도 입력 속도보다 기준을 맞추고 다시 확인하는 과정이 전체 업무 흐름을 안정적으로 만든다고 느꼈습니다.

이 프로젝트는 이러한 경험을 바탕으로 공개 이커머스 데이터를 활용해 운영 데이터의 정합성 문제를 분석한 프로젝트입니다. 실제 회사 내부 데이터는 사용하지 않고 주문·결제·배송·판매자 데이터의 연결 구조를 바탕으로 운영자가 확인해야 할 오류 후보를 정의했습니다.

## 3. 문제 정의

플랫폼 서비스에서는 주문, 결제, 배송, 판매자 정보가 각각 다른 테이블에서 관리됩니다. 이 데이터들이 정확하게 연결되지 않으면 고객 안내, 배송 추적, 결제 확인, 정산 검수 과정에서 혼선이 생길 수 있습니다.

예를 들어 주문은 존재하지만 결제 정보가 없거나, 배송 완료 상태인데 고객 수령일이 비어 있거나, 결제 금액과 상품 가격 및 배송비 합계가 크게 다르면 운영자가 다시 확인해야 할 가능성이 있습니다.

따라서 본 프로젝트에서는 주문·결제·배송·판매자 데이터를 연결해 정합성 점검 기준을 정의하고 SQL과 Python을 활용해 오류 후보를 탐지하고 검수 우선순위를 도출하고자 합니다.

## 4. 분석 목적

1. 주문·결제·배송·판매자 데이터 간 정합성 기준을 정의합니다.
2. 결제 누락, 상태값 불일치, 날짜 역전, 금액 불일치, 배송 지연 등 오류 후보를 탐지합니다.
3. 오류 유형별 발생 현황을 분석합니다.
4. 판매자별 오류율을 확인해 우선 검수 대상을 식별합니다.
5. 빈도, 영향도, 긴급도를 기준으로 검수 우선순위를 산정합니다.
6. 반복적인 운영 검수 업무에 활용할 수 있는 SQL 점검 로직을 정리합니다.
7. 배송 지연 여부와 리뷰 점수 차이를 보조적으로 확인합니다.
8. 금액 불일치 상위 사례를 별도로 추출해 검수 후보의 형태를 확인합니다.

## 5. 핵심 분석 질문

1. 주문·결제·배송 데이터에서 어떤 정합성 오류 후보가 발생하는가?
2. 오류 유형별 발생 빈도와 비중은 어떻게 다른가?
3. 어떤 오류 유형이 운영상 먼저 확인되어야 하는가?
4. 판매자별 오류율을 통해 우선 확인 대상을 정할 수 있는가?
5. 배송 지연 여부에 따라 리뷰 점수 차이가 나타나는가?
6. 금액 불일치 상위 사례는 어떤 형태로 나타나는가?
7. 이 분석 결과를 실제 운영 검수 기준으로 어떻게 활용할 수 있는가?

## 6. 기대 효과

이 프로젝트를 통해 운영 데이터 품질을 관리하기 위한 기준을 정의하고 SQL/Python으로 반복 가능한 점검 로직을 구현했습니다. 또한 공개 데이터의 한계를 고려해 실제 오류를 단정하지 않고 추가 확인이 필요한 검수 후보와 우선순위를 정리하는 방식으로 분석 결과를 해석했습니다.
