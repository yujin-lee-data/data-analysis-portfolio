# 지표 정의서

> 이 문서는 서비스 운영 데이터 품질 관리 프로젝트에서 사용한 핵심 지표와 계산·해석 기준을 정리합니다.

## 1. 지표 설계 목적

본 프로젝트의 지표는 공개 Olist 데이터 기준으로 운영 데이터 검수 후보를 정리하기 위해 정의했습니다. 결과는 실제 오류 확정이나 개선 성과를 의미하지 않으며 추가 확인이 필요한 후보를 우선순위화하기 위한 기준입니다.

## 2. 핵심 지표

| 지표 | 정의 | 해석 기준 |
|---|---|---|
| 주문/결제 정합성 | 주문 상태와 결제 정보 연결 여부를 비교한 기준 | 결제 정보가 없거나 연결 구조가 맞지 않는 주문을 검수 후보로 확인 |
| 금액 불일치 후보 | 주문 상품 금액과 배송비 합계가 결제 금액과 크게 다른 건 | 결제·정산 신뢰도 측면에서 우선 확인 필요 |
| 배송 지연 후보 | 예상 배송일과 실제 고객 수령일을 비교해 지연 여부를 구분한 건 | 고객 경험 저하 가능성을 확인하기 위한 참고 지표 |
| 판매자별 점검 후보 비율 | 판매자 단위로 지연·금액 차이 후보를 집계한 비율 | 특정 판매자에 오류 후보가 집중되는지 확인 |
| 우선순위 점수 | 빈도, 영향도, 긴급도를 종합한 검수 순서 보조 점수 | 실제 기업 정책이 아니라 본 프로젝트에서 정의한 분석용 기준 |

## 3. 계산 기준

| 항목 | 계산 또는 판정 기준 |
|---|---|
| 오류 후보 건수 | 오류 유형별 조건에 해당하는 주문 수 또는 사례 수 |
| 오류 후보 비율 | 오류 후보 건수 / 전체 기준 건수 |
| 판매자별 오류율 | 판매자별 오류 후보 건수 / 판매자별 주문 건수 |
| 우선순위 점수 | 빈도, 영향도, 긴급도를 가중해 산정한 검수 순서 참고 점수 |

## 4. 해석 유의사항

- 공개 데이터만으로 실제 오류 여부를 확정하지 않습니다.
- 결제 정책, 쿠폰·포인트·환불 정책, 배송 정책 등 내부 기준이 없으므로 결과는 검수 후보로만 해석합니다.
- 우선순위 점수는 실제 기업 정책이 아니라 분석 결과를 정리하기 위한 보조 기준입니다.
