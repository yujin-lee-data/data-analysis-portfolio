# 분석 기획서

> 이 문서는 프로젝트의 문제 정의, 분석 목적, 분석 범위와 기대 산출물을 정리합니다.

## 1. 문제 정의
이커머스 서비스에서 사용자는 여러 행동 단계를 거쳐 구매에 도달합니다. 본 프로젝트는 GA4 공개 이벤트 로그를 기준으로 대표 퍼널을 정의하고 어느 단계에서 전환이 약해지는지 확인하는 것을 목표로 합니다.

## 2. 분석 범위
- session_start
- view_item
- add_to_cart
- begin_checkout
- purchase

## 3. 분석 질문
1. 각 퍼널 단계의 사용자 수와 전환율은 어떤가?
2. 직전 단계 대비 이탈률이 높은 구간은 어디인가?
3. 디바이스, 카테고리, 구매자 여부에 따라 행동 차이가 관찰되는가?
4. 본 데이터 기준으로 우선 점검할 서비스 구간은 어디인가?

## 4. 해석 원칙
- 관찰 데이터 기준으로만 해석합니다.
- 실제 개선 결과나 인과관계는 단정하지 않습니다.
- 사용자 기준 집계와 이벤트 기준 집계를 구분합니다.
