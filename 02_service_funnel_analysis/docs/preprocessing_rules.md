# 전처리 기준서

> 이 문서는 원본 데이터 처리 원칙, 전처리 기준과 재현 시 유의사항을 정리합니다.

## 1. 전처리 원칙

| 기준 | 내용 |
|---|---|
| 분석 대상 이벤트 | `session_start`, `view_item`, `add_to_cart`, `begin_checkout`, `purchase`로 제한 |
| 전환율 기준 | 사용자 수 기준으로 계산 |
| 이벤트 수 활용 | 행동 빈도 참고 지표로만 사용 |
| 카테고리 분석 | `item_category`가 확인되는 이벤트만 활용 |
| 결측·미적재 항목 | 별도 해석 주의 대상으로 표시 |

## 2. SQL 처리 기준

공개 GA4 데이터셋은 이벤트 파라미터와 item 정보가 반복 구조로 저장될 수 있으므로 SQL 작성 시 UNNEST 처리 기준을 일관되게 적용했습니다. 퍼널 분석에서는 분석 단계 간 비교 가능성을 위해 동일한 사용자 식별 기준을 사용했습니다.

## 3. 재현 시 유의사항

BigQuery 공개 데이터에서 SQL 결과 CSV를 내려받은 뒤 Python 스크립트로 차트를 재생성하는 구조입니다. 따라서 BigQuery 원천 테이블에 직접 접근하지 않는 경우에도 저장된 CSV를 기준으로 시각화 결과를 확인할 수 있습니다.
