# 분석 기획서

> 이 문서는 프로젝트의 문제 정의, 분석 목적, 분석 범위와 기대 산출물을 정리합니다.

## 1. 문제 정의
커머스 서비스에서 매출은 단일 지표가 아니라 고객군, 재구매, 상품군, 쿠폰/캠페인 반응이 함께 만든 결과입니다. 본 프로젝트는 공개 거래 데이터를 기반으로 어떤 고객군과 상품군을 우선 관리해야 하는지 도출하는 것을 목표로 합니다.

## 2. 프로젝트 차별점
- 1번 프로젝트: 운영 데이터 품질/정합성 검수
- 2번 프로젝트: 서비스 퍼널/전환 분석
- 3번 프로젝트: 고객·매출·재구매·프로모션 반응 기반 비즈니스/커머스 분석

## 3. 분석 흐름
1. 거래 데이터 구조와 기간 확인
2. 장바구니 단위 구매 지표 생성
3. 고객별 매출 기여도와 재구매 행동 분석
4. 상품군별 매출 구조 분석
5. 쿠폰 사용 고객과 비사용 고객의 관찰상 차이 분석
6. 캠페인별 쿠폰 응답률 분석
7. 관리 우선순위와 한계점 정리

## 4. 해석 원칙
- 공개 데이터 기준으로만 설명합니다.
- 실제 기업 내부 결과처럼 표현하지 않습니다.
- 쿠폰/캠페인 분석은 인과효과가 아니라 관찰 데이터 기준 반응 차이로 표현합니다.
- DAY/WEEK_NO는 상대적 기간 변수로 명시합니다.
