# 전처리 기준서

> 이 문서는 원본 데이터 처리 원칙, 전처리 기준과 재현 시 유의사항을 정리합니다.

## 1. 전처리 기준
1. 컬럼명은 Python 분석에서 소문자 중심으로 표준화했습니다.
2. `SALES_VALUE`, `QUANTITY`, `DAY`, `WEEK_NO`, 할인 관련 컬럼은 숫자형으로 변환했습니다.
3. 고객 ID, 장바구니 ID, 상품 ID, 매출값, 기간 값이 없는 행은 핵심 분석에서 제외했습니다.
4. `SALES_VALUE`가 음수인 행은 매출 구조 분석에서 제외했습니다.
5. 할인 금액은 원본 데이터에서 음수로 기록된 값을 양수 금액으로 변환해 해석했습니다.
6. 원본 raw 데이터는 용량과 출처 관리를 위해 GitHub에 포함하지 않았습니다.

## 2. 재현성
`scripts/run_analysis.py`를 실행하면 `outputs/csv`와 `outputs/charts`가 재생성됩니다.
