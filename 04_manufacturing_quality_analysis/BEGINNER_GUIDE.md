# 초보자 실행 가이드

> 이 문서는 반도체 공정 품질 신호 분석 프로젝트를 처음 여는 사람도 실행 흐름을 이해할 수 있도록 정리한 안내서입니다.

## 1. 이 프로젝트가 하는 일

UCI SECOM 공개 데이터를 활용해 반도체 제조 공정 측정값의 결측률, Pass/Fail 그룹 차이, 변동성을 확인하는 프로젝트입니다.
분석의 목적은 불량 원인을 확정하는 것이 아니라 공개 데이터 기준으로 추가 점검이 필요한 품질 신호 후보를 정리하는 것입니다.

SECOM 데이터는 실제 공정명과 센서명이 익명화되어 있기 때문에 각 변수가 어떤 장비나 공정을 의미하는지는 알 수 없습니다. 따라서 분석 결과는 원인 규명이 아니라 점검 우선순위 후보 도출 관점으로 해석해야 합니다.

## 2. 필요한 준비물

* Python 3.10 이상
* 인터넷에서 패키지를 설치할 수 있는 환경
* `data/raw/uci-secom.csv` 파일

이 프로젝트에는 실행에 필요한 `uci-secom.csv` 파일이 포함되어 있어 바로 실행할 수 있습니다.

## 3. 데이터 구조 이해하기

이 데이터는 반도체 공정에서 측정된 여러 신호와 최종 품질 라벨로 구성되어 있습니다.

* 행: 공정 측정 사례
* 변수: 익명화된 공정 측정값
* 라벨: `Pass/Fail`
* `Pass`: 정상으로 분류된 사례
* `Fail`: 불량으로 분류된 사례

다만 Fail 비율이 낮은 불균형 데이터이므로 단순 정확도만으로 모델이나 분석 결과를 해석하지 않습니다.

## 4. 데이터 준비 방법

`data/raw/` 폴더에 아래 파일이 있는지 확인합니다.

```text
uci-secom.csv
```

파일이 있다면 별도 다운로드 없이 바로 실행할 수 있습니다.

## 5. 패키지 설치

터미널 또는 명령 프롬프트에서 프로젝트 폴더로 이동한 뒤 아래 명령어를 입력합니다.

```bash
pip install -r requirements.txt
```

## 6. 실행 순서

아래 명령어를 입력합니다.

```bash
python scripts/run_analysis.py
```

Windows에서 `python` 명령어가 실행되지 않으면 아래 명령어를 사용할 수 있습니다.

```bash
py scripts/run_analysis.py
```

Mac에서 실행한다면 아래 명령어를 사용할 수 있습니다.

```bash
python3 scripts/run_analysis.py
```

## 7. 결과 확인 방법

실행이 끝나면 아래 폴더를 확인합니다.

* 표 결과: `outputs/csv`
* 차트 이미지: `outputs/figures`
* 최종 PDF 보고서: `reports/04_manufacturing_quality_analysis_report.pdf`
* 최종 PPT 보고서: `reports/04_manufacturing_quality_analysis_report.pptx`

분석 과정에서 필요한 경우 SQLite DB가 로컬에 생성될 수 있습니다. DB 파일은 실행 결과물이며 GitHub 업로드 대상에는 포함하지 않습니다.

## 8. 주요 결과를 읽는 방법

이 프로젝트의 결과는 크게 네 가지 흐름으로 보면 됩니다.

1. 결측률 분석
   일부 공정 신호에 결측이 많아 데이터 품질 관리가 필요한 변수를 확인합니다.

2. Pass/Fail 그룹 차이 분석
   Pass 그룹과 Fail 그룹 사이에서 값의 차이가 크게 나타나는 신호를 확인합니다.

3. 변동성 분석
   값의 퍼짐이 크거나 극단값 가능성이 있는 신호를 확인합니다.

4. 점검 우선순위 도출
   결측, 그룹 차이, 변동성, 극단값 정보를 종합해 추가 점검이 필요한 품질 신호 후보를 정리합니다.

## 9. 자주 헷갈릴 수 있는 부분

* 파일 경로 오류: `data/raw/uci-secom.csv` 파일이 있는지 확인합니다.
* 패키지 설치 오류: `pip install -r requirements.txt`를 다시 실행합니다.
* 한글 글꼴 표시 오류: 분석 결과에는 영향이 없으며 실행 환경에 따라 차트 글꼴이 다르게 보일 수 있습니다.
* DB 파일 관련 오류: 기존에 생성된 DB 파일이 충돌할 경우 삭제 후 다시 실행하면 됩니다.

## 10. 해석 시 주의사항

이 프로젝트는 UCI SECOM 공개 데이터를 활용한 관찰 분석입니다. 변수명이 익명화되어 실제 센서나 공정 의미를 알 수 없으므로 분석 결과는 불량 원인 판단이 아니라 추가 점검이 필요한 품질 신호 후보로 해석했습니다.

Pass/Fail 그룹 차이나 변동성이 크게 나타난 신호도 불량의 원인이라고 단정할 수 없습니다. 실제 현장 해석에는 공정 지식, 설비 정보, 센서 의미, 추가 검증이 필요합니다.

모델 결과는 참고용 보조 지표이며 실제 현장 적용 성능이나 불량률 감소를 의미하지 않습니다.
