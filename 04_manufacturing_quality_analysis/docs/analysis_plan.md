# 분석 기획서

> 이 문서는 프로젝트의 문제 정의, 분석 목적, 분석 범위와 기대 산출물을 정리합니다.

## 1. 프로젝트 개요

- 프로젝트명: Semiconductor Process Quality Signal Analysis
- 한국어명: 반도체 제조 공정 품질 신호 분석 및 점검 우선순위 도출
- 폴더명: `04_manufacturing_quality_analysis`
- 데이터: Kaggle UCI SECOM Dataset (`uci-secom.csv`)
- 목적: 반도체 제조 공정 공개 데이터를 활용해 Pass/Fail 라벨과 공정 측정값의 관계를 관찰하고 추가 점검이 필요한 품질 신호 후보를 우선순위화합니다.

> 해석 원칙: 본 프로젝트는 공개 데이터 기준의 관찰 분석입니다. 실제 제조 공정의 구체적 판단을 확정하거나 성과를 검증하지 않습니다. 결과는 우선 점검 후보를 정리하기 위한 참고 자료이며 현장 해석에는 공정 지식과 추가 검증이 필요합니다.

## 2. 문제 정의

반도체 제조 공정은 다수의 센서 및 측정값으로 관리됩니다. 하지만 공정 측정 신호가 많을수록 모든 항목을 동일한 수준으로 검토하기 어렵습니다. 따라서 결측률, Pass/Fail 그룹 간 분포 차이, 측정값 변동성을 기준으로 우선 점검할 신호 후보를 정의하는 과정이 필요합니다.

본 프로젝트는 SECOM 공개 데이터를 활용해 다음 질문에 답합니다.

1. 공개 데이터 기준 Pass/Fail 라벨은 어떤 분포를 보이는가?
2. 결측률이 높은 공정 측정 신호는 무엇인가?
3. Pass와 Fail 그룹 간 차이가 큰 측정 신호는 무엇인가?
4. 변동성이 크거나 극단값 가능성이 높은 측정 신호는 무엇인가?
5. 위 기준을 종합했을 때 우선 점검 후보로 볼 수 있는 신호는 무엇인가?

## 3. 분석 범위

- 포함: 데이터 구조 점검, 결측률 분석, 라벨 분포 분석, 그룹 차이 분석, 변동성 분석, 우선순위 점수 산정, 시각화, SQL 기반 요약
- 선택: 단순 기준 모델을 통한 참고용 변수 중요도 확인
- 제외: 실제 원인 확정, 실제 성과 검증, 현장 적용 가능성 주장, 인과효과 주장

## 4. 분석 흐름

1. 원본 데이터 적재
2. 변수명 정리: `0~589` 컬럼을 `feature_000~feature_589`로 변경
3. 라벨명 생성: `-1=Pass`, `1=Fail`
4. 결측률 요약
5. Pass/Fail 분포 확인
6. Pass/Fail 그룹별 평균 및 중앙값 차이 계산
7. 표준화 평균 차이 산출
8. 표준편차, IQR, IQR 기준 극단값 비율 계산
9. 점검 우선순위 점수 산정
10. 결과 CSV와 차트 저장
11. SQLite DB 생성 후 SQL 쿼리 재현

## 5. 최종 산출물

## 6. 활용 가능 관점

- 제조/품질 데이터에서 결측률, 그룹 차이, 변동성을 기준으로 점검 후보를 정리하는 방식으로 활용할 수 있습니다.
- 변수명이 익명화된 공개 데이터의 한계를 고려해 실제 원인 판단이 아니라 우선 확인이 필요한 신호를 정리하는 관점으로 해석합니다.
- SQL/Python으로 결과 CSV, 차트, SQLite 요약 테이블을 재현 가능하게 정리했습니다.
