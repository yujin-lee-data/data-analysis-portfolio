# 데이터 정의서

> 이 문서는 분석에 사용한 데이터 구조, 주요 컬럼과 해석 유의사항을 정리합니다.

## 1. 데이터 개요

- 파일명: `uci-secom.csv`
- 출처: Kaggle UCI SECOM Dataset
- 주제: Semiconductor manufacturing process dataset
- 행 수: 1,567개
- 열 수: 592개
- 주요 구성: 측정 시점, 익명화된 공정 측정 변수, Pass/Fail 라벨

> 해석 원칙: 본 프로젝트는 공개 데이터 기준의 관찰 분석입니다. 실제 제조 공정의 구체적 판단을 확정하거나 성과를 검증하지 않습니다. 결과는 우선 점검 후보를 정리하기 위한 참고 자료이며 현장 해석에는 공정 지식과 추가 검증이 필요합니다.

## 2. 원본 컬럼 구조

| 컬럼 | 설명 | 본 프로젝트 처리 |
|---|---|---|
| `Time` | 측정 시점 | 날짜형으로 변환, 원본 보존 |
| `0`~`589` | 익명화된 반도체 제조 공정 측정값 | `feature_000`~`feature_589`로 변경 |
| `Pass/Fail` | 최종 품질 라벨 | `-1=Pass`, `1=Fail`로 해석 |

## 3. 분석용 컬럼

| 컬럼 | 설명 |
|---|---|
| `Time` | 측정 시점 |
| `feature_000`~`feature_589` | 익명화된 공정 측정 신호 |
| `Pass/Fail` | 원본 라벨. -1은 Pass, 1은 Fail |
| `label_name` | 분석 편의를 위해 만든 라벨명. Pass 또는 Fail |

## 4. 라벨 분포

| 라벨 | 의미 | 건수 | 비율 |
|---|---|---:|---:|
| -1 | Pass | 1,463 | 93.36% |
| 1 | Fail | 104 | 6.64% |

## 5. 데이터 사용상 주의사항

1. 공정 변수는 익명화되어 있어 실제 센서명이나 공정 단계를 알 수 없습니다.
2. Fail 라벨은 실제 공정상 구체적 판단 확정값이 아니라 제공 데이터 기준의 품질 라벨입니다.
3. 결측치가 존재하므로 결측률 자체도 관리 리스크 신호로 해석합니다.
4. Fail 사례가 Pass보다 훨씬 적어 클래스 불균형이 존재합니다.
5. 모델을 사용하더라도 현장 적용 가능성을 주장하지 않습니다.
