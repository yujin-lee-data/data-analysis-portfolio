# 전처리 기준서

> 이 문서는 원본 데이터 처리 원칙, 전처리 기준과 재현 시 유의사항을 정리합니다.

> 해석 원칙: 본 프로젝트는 공개 데이터 기준의 관찰 분석입니다. 실제 제조 공정의 구체적 판단을 확정하거나 성과를 검증하지 않습니다. 결과는 우선 점검 후보를 정리하기 위한 참고 자료이며 현장 해석에는 공정 지식과 추가 검증이 필요합니다.

## 1. 원본 데이터 보존

- `data/raw/uci-secom.csv`는 수정하지 않습니다.
- 분석 과정에서 생성되는 주요 결과 파일은 `outputs/csv`와 `outputs/figures`에 저장합니다.
- 실행 시 SQLite DB가 `database/secom_analysis.sqlite`로 생성될 수 있으나 DB 파일은 로컬 실행 결과물이며 GitHub 업로드 대상에는 포함하지 않습니다.

## 2. 변수명 변경

원본 측정 변수는 `0~589` 숫자 컬럼명으로 되어 있습니다. 분석 문서와 SQL에서 다루기 쉽도록 다음과 같이 변경합니다.

| 원본 | 변경 후 |
|---|---|
| `0` | `feature_000` |
| `1` | `feature_001` |
| ... | ... |
| `589` | `feature_589` |

## 3. 날짜 처리

- `Time` 컬럼은 `datetime`으로 변환합니다.
- 변환 실패값은 결측으로 둡니다.
- 본 프로젝트의 핵심 분석은 시계열 모델링이 아니라 품질 신호 후보 도출이므로 시간은 보조 정보로만 사용합니다.

## 4. 라벨 처리

- `Pass/Fail = -1`: Pass
- `Pass/Fail = 1`: Fail
- 분석 편의를 위해 `label_name` 컬럼을 추가합니다.

## 5. 결측치 처리

- 결측치는 바로 제거하지 않습니다.
- 먼저 변수별 결측률을 계산합니다.
- 결측률 40% 이상 변수는 해석 주의 대상으로 표시합니다.
- 그룹 차이와 우선순위 분석에서는 결측을 제외한 관측값 기준으로 통계량을 계산합니다.
- 선택 모델에서는 중앙값 대체를 사용합니다.

## 6. 극단값 처리

- 본 프로젝트는 극단값을 임의 삭제하지 않습니다.
- IQR 기준 범위 밖 값 비율을 계산해 변동성이 큰 품질 신호 후보를 확인하는 참고 지표로 사용합니다.

## 7. 모델링 기준

- 모델은 선택 분석입니다.
- 모델 성능은 현장 적용 가능성을 주장하기 위한 지표가 아닙니다.
- 목적은 품질 라벨 구분에 참고가 되는 측정 신호 후보를 추가로 확인하는 것입니다.
